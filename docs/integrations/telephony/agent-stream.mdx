## Overview

Agent Stream is a generic WebSocket endpoint reserved for provider-specific
media-stream adapters:

```
wss://your-backend.example.com/api/v1/agent-stream/{agent_uuid}
```

The endpoint validates the agent UUID, checks quota, creates a workflow run,
and dispatches to the telephony provider named by the `provider` query
parameter.

<Warning>
  The providers included in Dograh currently do not implement Agent Stream.
  Calls to this endpoint with an unsupported provider return `1011`.
</Warning>

## Query Parameters

| Param | Required | Description |
| --- | --- | --- |
| `provider` | Yes | Registered telephony provider name. |
| `callId` / `CallSid` | No | Call identifier persisted on the workflow run's gathered context. |
| `from` | No | Caller phone number persisted as `caller_number`. |
| `to` | No | Called phone number persisted as `called_number`. |

## Security Notes

- Treat the URL as a secret because the agent UUID itself authorizes the connection.
- Use TLS and avoid logging full URLs with query parameters.
- Provider implementations must load credentials server-side from Dograh configuration, not from public query parameters.
